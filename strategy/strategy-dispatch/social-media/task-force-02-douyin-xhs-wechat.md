# Task Force 02 — China Growth Trinity（抖音 + 小红书 + 微信公众号）

**使命**：在华语主战场用**趋势信号 → 可执行内容 → 私域/转化衔接**闭环打穿算法分发；三平台**同一战略、不同表达**，禁止简单搬运。

---

## Playbook：爆款样本采集 → 流量密码拆解 → 基于素材撰稿（Sprint 0）

> **定位**：每次「日报 / 周报 / 战役周」正式写稿前，必须先跑一遍 **Sprint 0（情报闭环，约 2–4h，可按周固化）**。  
> **组长负责**：China Market Localization Strategist 定「采哪些领域、采多少篇、密码卡片长什么样」；**Douyin / Xiaohongshu / WeChat OA** 三专家各认领本台样本池；**Content Creator** 把密码翻译成可写 brief；**Brand Guardian** 划「借鉴 vs 抄袭」红线。

### 0）界定「相关领域」（15 min）

产出一张 **领域界定表**（可放进周 handoff）：

| 项 | 填什么 |
|----|--------|
| 赛道/话题簇 | 例：AI 工具、职场效率、母婴、本地生活…（一次不超过 **3 个簇**） |
| 受众画像 | 年龄带、城市线、决策角色（老板/采购/学生） |
| 竞品/对标账号池 | 每平台 **5–15 个**活跃账号（含 2 个「跨圈黑马」对照组） |
| 关键词矩阵 | 每簇 **10–30 个**词：大词、长尾、痛点词、避雷词 |
| 本周禁忌 | 品牌红线 + 平台敏感词（医疗金融等另表） |

### 1）爆款样本「采集」——优先合规渠道（禁止默认非法爬站）

口语里的「爬爆款」在本 playbook 中统一解释为：**系统性收集高表现样本并结构化存档**。  
**未经平台授权、绕过登录、批量抓取正文/评论/私信**等做法，可能违反服务条款并带来法务风险；**默认禁止**。可接受路径如下（按优先级）：

| 方式 | 平台 | 做法 | 负责倾向 |
|------|------|------|----------|
| **平台原生热榜/创作灵感/搜索下拉** | 抖音、小红书 | 官方页面人工摘录 + 截图归档 | Douyin / XHS |
| **对标账号主页—人工点选近期高赞** | 抖音、小红书、视频号 | 每条样本记录链接、发布时间、互动量级（粗估即可） | 三专家分工 |
| **微信生态** | 公众号、搜一搜 | 用微信客户端/搜一搜检索关键词，**人工**打开文章→复制链接→粘贴到团队表格；可用壹伴/新榜等**已签约**工具导出榜单（按供应商合规指引） | WeChat OA Manager |
| **第三方数据产品** | 多平台 | 飞瓜/灰豚/新榜等 **API 或导出报表**（合同范围内使用） | 组长 + Analytics |
| **自动化脚本** | 任意 | **仅**在具备书面授权、且法务已审 ToS 的前提下启用；否则不启动 | 组长 + Legal |

**最低样本量（建议，可据资源调整）**

- 抖音：近 **7–14 天**内，同赛道 **≥15 条**「高完播/高赞」短视频（含 3 条「翻车/争议」对照）。  
- 小红书：近 **14 天**内，同赛道 **≥20 篇**笔记（含封面/标题/正文首屏截图）。  
- 微信公众号：近 **30 天**内，同赛道 **≥10 篇**「高阅读或高在看」图文（链接 + 标题 + 首段 + 转发语风格）。

### 2）单篇拆解：把「爆款」压成「流量密码字段」

每条样本填一张 **拆解表**（飞书/Notion/表格均可），至少包含：

| 字段 | 写什么 |
|------|--------|
| 样本 URL / 截图 | 可回溯 |
| 平台与体裁 | 短视频 / 图文 / 系列连载 |
| **钩子** | 前 3 秒 / 首图 / 标题前半句，到底勾了什么情绪（恐惧、爽感、优越感、省钱、吃瓜） |
| **信息结构** | 清单体 / 故事弧 / 对话体 / 对比表格… |
| **密度与节奏** | 几秒一个反转；段落是否「一句一段」 |
| **互动设计** | 评论区引导、扣 1、争议点、投票、文末互动句式 |
| **变现/导流** | 是否挂车、是否导私域、话术是否「软」 |
| **可复用句式** | 直接摘 3 句「可改写模板」（勿原句抄袭发布） |
| **风险标签** | 夸大、引战、医疗功效…（供 Brand Guardian 秒拒） |

### 3）聚合成「流量密码卡片」（组长 Gate）

把拆解表中重复出现的结构，抽象成 **≤7 张密码卡片**，每张卡固定格式：

- **名称**：如「封面大字 + 两行痛点」「前 3 秒先骂自己再反转」  
- **适用平台**：抖音 / 小红书 / 公号（可多选）  
- **可验证假设**：若采用该结构，预期提升的指标（完播 / 收藏 / 打开率）  
- **本周实验次数**：例如抖音测 2 条、小红书测 2 篇  
- **素材依赖**：需要哪些事实素材、是否已齐（不齐则先补采）

**Gate B-2（二部专用）**：未通过 Gate B-2（密码卡片齐 + 样本表齐）**不得进入撰稿**。

### 4）基于素材撰稿（与总 playbook 第 4 节对齐）

```
流量密码卡片 + 领域事实素材（新闻/产品/数据）
    → Content Creator：母本（长逻辑 + 金句 + 禁忌检查清单）
    → Douyin Strategist / Xiaohongshu Specialist / WeChat OA Manager：本台改写（钩子、时长、封面、菜单/摘要）
    → Brand Guardian：Gate C 签核
    → 组长：排期 + 首小时互动预案
```

**原创与借鉴红线**

- **结构可借鉴，表达须原创**；禁止大段洗稿、盗图、未授权搬运。  
- 涉及第三方报道与数据：**可追溯链接 + 编辑部核对**后再写进终稿。

### 5）复盘：把「密码」接回数据

发布后 **48–72h**，Analytics 将每条内容与所用「密码卡片」编号关联，复盘：

- 哪张卡在哪个平台验证/证伪；  
- 下周样本池是否换一批对标（避免永远抄同一账号）。

## 与 TF-01 的交叉规则

- **Content Creator、Brand Guardian、Analytics Reporter** 为首选共享池；**Trend Researcher** 可提供全球结构灵感，但**华语热榜与评论挖掘**以组长为主导。  
- 同一产品全球发布：**两条 TF 各出一份平台化叙事**，由客户方指定「主轴语言」后再翻译改写，避免英文直译腔。
